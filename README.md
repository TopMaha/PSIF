## ทำให้เป็นฐานข้อมูลออนไลน์ (ทุกคนเห็นข้อมูลเดียวกัน)

> เครื่องนี้ deploy Worker ไม่ได้ (Node เก่า/ไม่มี wrangler) — ทำตามขั้นตอนด้านล่างบนเครื่องที่มี
> Node 18+ หรือทำผ่านหน้าเว็บ Cloudflare Dashboard

### A. ติดตั้ง wrangler (เครื่องที่ deploy)
```bash
npm install -g wrangler
wrangler login
```

### B. สร้าง D1 + R2
```bash
wrangler d1 create psif-db
wrangler r2 bucket create psif-photos
wrangler d1 execute psif-db --remote --file=./schema.sql
```
คัดลอก `database_id` ที่ได้จากคำสั่งแรกไปใส่ `wrangler.toml`

### C. สร้าง `wrangler.toml` (วางไว้โฟลเดอร์เดียวกับ `worker.js`)
```toml
name = "psif-api"
main = "worker.js"
compatibility_date = "2024-09-01"

[[d1_databases]]
binding = "DB"
database_name = "psif-db"
database_id = "<<วาง database_id ที่ได้จากขั้น B>>"

[[r2_buckets]]
binding = "BUCKET"
bucket_name = "psif-photos"

# (ไม่บังคับ) ถ้าเปิด Public access / Custom domain ให้ R2 แล้ว
# ใส่ URL ฐานไว้ตรงนี้ เพื่อให้รูปโหลดตรงจาก R2 (ลดภาระ Worker)
# [vars]
# R2_PUBLIC_BASE = "https://pub-xxxxxxxx.r2.dev"
```

### D. Deploy
```bash
wrangler deploy
```
จะได้ URL เช่น `https://psif-api.<ชื่อบัญชี>.workers.dev`

### E. เชื่อมแอปกับ API
เปิดแอป → เมนู **⚙️ ตั้งค่า → ฐานข้อมูลออนไลน์ (API)** → วาง URL → **บันทึก & เชื่อมต่อ**
ทุกคนที่ใส่ URL เดียวกันจะเห็น/แก้ไขข้อมูลร่วมกัน

> **ทางเลือกผ่าน Dashboard (ไม่ใช้ command line):** Cloudflare → Workers & Pages → Create Worker →
> วางโค้ด `worker.js` → ในแท็บ Settings ผูก Binding ชื่อ `DB` (D1) และ `BUCKET` (R2) →
> รัน `schema.sql` ในหน้า D1 Console → Deploy

## 📷 การเก็บรูปภาพ (ประหยัดพื้นที่)
- **ไฟล์รูปจริงเก็บบน Cloudflare R2** — ฐานข้อมูล D1 เก็บแค่ "คีย์" (ข้อความสั้น เช่น
  `psif/12/after-….jpg`) ไม่ได้เก็บไฟล์รูป → D1 เล็กมาก ไม่บวม
- **ย่อรูปอัตโนมัติฝั่งผู้ใช้** ก่อนอัปโหลด (ด้านยาว ≤ 1600px, JPEG q0.82 ≈ 250–420 KB/รูป)
  ปรับความชัด/ขนาดได้ที่ตัวแปร `PHOTO_MAX` และ `PHOTO_QUALITY` ใน `index.html`
  - R2 ฟรี 10 GB ≈ ~12,000 งาน (งานละ 2 รูป) — อยากชัดขึ้นเพิ่มเป็น 1920px ได้ (ไฟล์โตขึ้นเล็กน้อย)
  - D1 เก็บแค่คีย์รูป จึงไม่มีทางทำให้ฐานข้อมูลเต็ม
- **ลบรูปออกจาก R2 อัตโนมัติ** เมื่อ Admin ลบรายการ (`DELETE /psif`)
- เสิร์ฟรูป 2 แบบ:
  1. ผ่าน Worker `GET /photo/:key` (ค่าเริ่มต้น — ใช้ได้เลย, รูปเป็นส่วนตัว)
  2. ตรงจาก R2 public/custom domain — ตั้ง `R2_PUBLIC_BASE` (ดูขั้น C) เพื่อลดภาระ/ค่า request ของ Worker
- โหมดทดลอง (ยังไม่ตั้งค่า API): รูปเก็บแบบ base64 ในเครื่อง (localStorage) ชั่วคราวเท่านั้น —
  พอเชื่อม API/deploy Worker แล้ว รูปจะไปอยู่บน R2 ทันที

## การทำงานของ 4 ขั้นตอน (ข้อ 1)
1. **บันทึกเข้าระบบ** — พนักงานกรอกฟอร์ม · แนบรูป "ก่อนแก้ไข" **บังคับเฉพาะประเภท PSIF (Con)**
   (ประเภทอื่นซ่อนช่องรูป ไม่ต้องแนบ — ตรวจซ้ำทั้ง frontend และ backend)
2. **Safety อนุมัติ/ไม่อนุมัติ** — เจ้าหน้าที่ Safety/Admin ตรวจเนื้อหา + ดูเรื่องซ้ำ
3. **กำลังดำเนินการ** — เริ่มได้เมื่อ **Admin "เปิดดำเนินการ"** ของหน่วยงานนั้น (ข้อ 6) แล้วใส่เลข No.PSIF
4. **เสร็จ** — ปิดงาน · แนบรูป "หลังแก้ไข" บังคับเฉพาะ PSIF (Con)

ระบบกัน **บันทึกซ้ำ** อัตโนมัติ: ปุ่มบันทึกล็อกทันทีที่กด + ทุกการบันทึกส่ง `requestId` (UUID)
ให้ Worker ตรวจซ้ำ (idempotency) และมี unique index ที่ D1 เป็นชั้นสุดท้าย
(DB ที่มีอยู่แล้วต้องรัน `migrate-2026-07-15-idempotency.sql` หนึ่งครั้ง)

## สิทธิ์ผู้ใช้ (เช็คจริงที่ Backend — role อ่านจากตาราง employees ผ่าน header `X-Emp-Id`)
- `user` — บันทึก/ปิดงานของตัวเอง (หน่วยงานระบุอัตโนมัติจากทะเบียนพนักงาน)
- `teamleader` / `manager` — เท่า user + ดูรายคนของกลุ่มตัวเองบนแดชบอร์ด
- `dept_admin` — **Admin แผนก**: เห็น/ติดตาม/ดำเนินการ (ออกเลข No., ปิดงาน, แก้เนื้อหา)
  เฉพาะแผนกตัวเอง (ช่องหน่วยงาน/VSM = แผนก, กรองที่ query ฝั่ง Worker) · Export Report ได้ ·
  **เข้าหน้าตั้งค่าไม่ได้**
- `safety` — อนุมัติ Safety + แก้ไขเนื้อหา + Export Report · **เข้าหน้าตั้งค่าไม่ได้**
- `admin` — **Super Admin**: ทุกแผนก + หน้าตั้งค่าทั้งหมด (ข้อมูลหลัก, เปิดดำเนินการ,
  เป้าหมาย, นำเข้า Excel, ลบรายการ)

## 📊 Report (หน้า "ติดตาม")
ปุ่ม **Report** ส่งออกไฟล์ **Excel (.xlsx)** รายละเอียดครบทุกฟิลด์ ตาม filter ที่เลือกอยู่บนหน้าจอ
สิทธิ์: Admin แผนก (เฉพาะแผนกตัวเอง) / Safety / Super Admin

## ตรรกะการนับ (KPI/แดชบอร์ด/โบนัส)
นับเป็น "ผลงาน" เฉพาะเรื่องที่เข้า **ทั้ง 2 เงื่อนไข**: Safety **อนุมัติ** และดำเนินการ**จนจบ (เสร็จ)**
นอกนั้นไม่นับทุกกรณี (การ์ด "ส่งแล้ว" และกราฟส่งรายเดือน เป็นเมตริกการส่ง แสดงประกอบเท่านั้น)

## โบนัส — คิดจากเรื่องที่ผ่านเกณฑ์การนับด้านบน แยกตามประเภท
ผ่าน PSIF(Con) ≥2 = **5%** พื้นฐาน · ครบ Near miss ≥2 **+2.5%** · ครบ PSIF(Behavior) ≥2 **+2.5%** (สูงสุด 10%)

ปรับเป้าหมาย/คน/ปี ได้ที่ เมนูตั้งค่า (Super Admin)
